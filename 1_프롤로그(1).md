# 🎬 [프롤로그] 대학교 데이터베이스 첫걸음 (수강신청 편)

데이터베이스의 세계에 오신 것을 환영합니다! 
우리는 방금 빈 테이블(그릇) 3개를 만들고, 그 안에 학생, 과목, 수강신청 내역이라는 진짜 데이터(음식)를 채워 넣었습니다. 

이제 이 대학교의 학사 관리자가 되었다고 생각하고, 엑셀의 필터나 정렬 기능을 쓰듯이 쿼리(Query)를 날려서 데이터를 조회하고 수정해 보는 맛보기 시간을 가져보겠습니다.

---

## 🔍 실습 1. 데이터 조회하기 (기초 SELECT)

가장 중심이 되는 `student`(학생) 테이블을 이리저리 요리해 보겠습니다. 쿼리를 실행할 때마다 결과가 어떻게 변하는지 눈으로 확인해 보세요.

### 1-1. 학생 테이블 전체 조회 (가장 기본!)
```sql
-- 대학교에 등록된 모든 학생의 모든 정보를 보여주세요.
SELECT * FROM student;
```

### 1-2. 보고 싶은 컬럼(칸)만 골라서 보기
```sql
-- 학번, 이름, 학과, 학년만 깔끔하게 추려서 보여주세요.
SELECT stu_no, stu_name, stu_dept, stu_grade 
FROM student;
```

### 1-3. 조건 달기 (WHERE): 특정 학과 학생만 찾기
```sql
-- '컴퓨터정보' 학과 학생들만 명단만 뽑아주세요.
SELECT stu_name, stu_dept, stu_grade 
FROM student 
WHERE stu_dept = '컴퓨터정보';
```

### 1-4. 복합 조건 (AND): 키와 몸무게로 검색하기
```sql
-- 키가 170 이상이면서(AND), 몸무게는 70 이하인 날렵한 학생을 찾아주세요.
SELECT stu_name, stu_height, stu_weight 
FROM student 
WHERE stu_height >= 170 AND stu_weight <= 70;
```

### 1-5. 정렬하기 (ORDER BY): 키 순서대로 줄 세우기
```sql
-- 학생들의 이름을 키가 큰 순서대로 내림차순(DESC) 정렬해 주세요.
SELECT stu_name, stu_dept, stu_height 
FROM student 
ORDER BY stu_height DESC;
```

### 1-6. 빈칸 찾기 (IS NULL): 신체검사 누락자 찾기
```sql
-- 키 정보가 아직 입력되지 않은(NULL) 학생을 찾아주세요. (박희철, 김종헌 학생이 나올 겁니다!)
SELECT stu_name, stu_dept, stu_height 
FROM student 
WHERE stu_height IS NULL;
```

## 🔄 실습 2. 데이터 수정과 삭제 (UPDATE & DELETE)

학생의 정보가 바뀌거나 수강을 취소했을 때, 데이터를 고치고 지워보겠습니다. 항상 `WHERE` 절(조건)을 빼먹지 않도록 주의하세요!

### 2-1. 수정 (UPDATE): 옥한빛 학생의 반 변경
```sql
-- 기계과 옥한빛 학생(학번: 20153075)이 열심히 공부해서 C반에서 A반으로 승급했습니다!
UPDATE student 
SET stu_class = 'A' 
WHERE stu_name = '옥한빛';

-- (확인용 쿼리) 옥한빛 학생의 반이 'A'로 바뀌었는지 확인해 보세요.
SELECT stu_name, stu_class FROM student WHERE stu_name = '옥한빛';
```

### 2-2. 삭제 (DELETE): 수강 취소 처리
```sql
-- 옥한빛 학생(학번: 20153075)이 '기계공작법(과목코드: 102)' 수강을 취소했습니다. 수강 내역(enrol)에서 지워주세요.
DELETE FROM enrol 
WHERE stu_no = '20153075' AND sub_no = '102';

-- (확인용 쿼리) 옥한빛 학생의 수강 내역에서 102번 과목이 사라졌는지 확인해 보세요.
SELECT * FROM enrol WHERE stu_no = '20153075';
```

## 💾 실습 3. 도장 쾅 찍기! (COMMIT)

지금까지 우리가 옥한빛 학생의 반을 바꾸고(`UPDATE`), 수강을 취소한(`DELETE`) 내역은 아직 우리 컴퓨터 메모리에만 임시로 존재하는 상태입니다. 

이 변경 사항을 대학교 메인 데이터베이스 하드디스크에 영구적으로 반영해서 다른 조교들도 볼 수 있게 확정해 봅시다.

```sql
COMMIT;
```
