# 🏃‍♂️ 2단계: SQL 기본기 다지기 (명령어 종류와 NULL 정복)

데이터베이스라는 거대한 식당을 운영하려면 다양한 지시를 내려야 합니다. 식탁(테이블)을 새로 만들기도 하고, 손님(데이터)을 자리에 앉히기도 하고, 알바생에게 금고 열쇠를 주기도 해야 하죠. 

SQL 명령어는 그 **'목적'에 따라 크게 4가지**로 나뉩니다. 이 4가지의 소속을 구분하는 문제가 무조건 1문제 이상 출제됩니다!

---

## 1. 🗂️ SQL 명령어의 4가지 종류 (반드시 암기!)

### 🏗️ DDL (Data Definition Language - 데이터 정의어)
* **역할:** 테이블(식탁)이나 인덱스 같은 '구조'를 만들고, 변경하고, 부수는 명령어입니다. **(건축 담당)**
* **명령어:** `CREATE`(생성), `ALTER`(수정), `DROP`(삭제), `TRUNCATE`(내용물 전체 삭제)
* **특징:** 실행하는 순간 DB에 즉시 반영되며(Auto Commit), 되돌릴 수 없습니다!

### 🧑‍🍳 DML (Data Manipulation Language - 데이터 조작어)
* **역할:** 만들어진 테이블 안에 실제 데이터(손님)를 넣고, 빼고, 수정하고, 조회하는 명령어입니다. **(서빙 담당)**
* **명령어:** `SELECT`(조회), `INSERT`(삽입), `UPDATE`(수정), `DELETE`(삭제)
* **특징:** 우리가 가장 많이 쓰는 명령어입니다. 실행 후에도 마음에 들지 않으면 `ROLLBACK`으로 되돌릴 수 있습니다.

### 👮 DCL (Data Control Language - 데이터 제어어)
* **역할:** 특정 사용자에게 권한을 주거나 뺏는 명령어입니다. **(보안 담당)**
* **명령어:** `GRANT`(권한 부여), `REVOKE`(권한 회수)

### 💾 TCL (Transaction Control Language - 트랜잭션 제어어)
* **역할:** DML(데이터 조작)로 일으킨 변화를 최종적으로 확정하거나, 취소하는 명령어입니다. **(결제 담당)**
* **명령어:** `COMMIT`(영구 저장 확정), `ROLLBACK`(마지막 COMMIT 이후의 작업 모두 취소)

## 2. 🔍 데이터를 골라내는 마법의 조건절 (`WHERE`)

수십만 건의 데이터 중 내가 원하는 데이터만 쏙쏙 뽑아내려면 `SELECT` 문 뒤에 `WHERE` 조건절을 잘 써야 합니다.

* **`IN (A, B, C)`:** "A이거나 B이거나 C인 사람 다 나와!" (OR 조건의 묶음입니다)
* **`BETWEEN A AND B`:** "A부터 B 사이에 있는 사람 다 나와!" (A와 B 값도 **포함**됩니다)
* **`LIKE` (문자열 검색):** 특정 패턴의 글자를 찾습니다.
  * `%` (퍼센트): 0개 이상의 모든 글자를 의미합니다. (`LIKE '김%'` -> 김으로 시작하는 모든 글자)
  * `_` (언더바): 딱 1개의 글자를 의미합니다. (`LIKE '김_'` -> 김으로 시작하는 딱 2글자 이름)

## 3. 👻 시험의 알파이자 오메가, `NULL` 완벽 정복

SQLD 수험생들을 가장 많이 울리는 녀석이 바로 이 **`NULL`**입니다. SQL에서 NULL은 0(숫자)도 아니고, 공백(' ')도 아닙니다. **"아직 값이 입력되지 않은, 모르는 값(Unknown)"**을 의미합니다.

### 🚨 NULL의 절대 규칙 3가지
1. **연산하면 전염병처럼 퍼진다:** `NULL + 10 = NULL` 입니다. 모르는 값에 10을 더해봤자 여전히 모르는 값이기 때문입니다.
2. **비교 연산자(=, >, <)로 찾을 수 없다:** `WHERE 나이 = NULL` 이라고 쓰면 데이터를 절대 찾을 수 없습니다. 반드시 **`WHERE 나이 IS NULL`** 또는 **`IS NOT NULL`** 이라고 써야 합니다!
3. **집계 함수는 NULL을 무시한다:** `SUM`, `AVG`, `COUNT(컬럼명)` 같은 함수들은 데이터를 계산할 때 NULL이 들어있는 칸을 아예 없는 취급하고 계산합니다.

### 🛠️ NULL을 다른 값으로 바꿔주는 치료 함수
* **`NVL(컬럼명, 대체값)`:** 해당 컬럼의 값이 NULL이면 '대체값'으로 바꿔서 출력합니다. (SQL Server에서는 `ISNULL`을 사용합니다.)
  * 예: `NVL(보너스, 0)` -> 보너스가 NULL인 사람은 0원으로 계산하라!
* **`COALESCE(A, B, C, ...)`:** 괄호 안의 인자들을 순서대로 살펴보고, **'가장 처음으로 NULL이 아닌 값'**을 출력합니다.

## 4. 🚨 초보자 단골 함정 (트러블슈팅 가이드)

### ❌ 함정 1: `DROP` vs `TRUNCATE` vs `DELETE` 의 차이점
이 세 가지 모두 무언가를 '지우는' 명령어라 시험에 아주 단골로 나옵니다.
* **`DELETE` (DML):** 조건(`WHERE`)을 줘서 원하는 데이터만 골라서 지울 수 있습니다. 잘못 지우면 `ROLLBACK`으로 복구 가능합니다. (가장 느리고 찌꺼기가 남음)
* **`TRUNCATE` (DDL):** 테이블의 껍데기(구조)는 남겨두고 안의 내용물만 영혼까지 싹 다 비워버립니다. 조건(`WHERE`)을 쓸 수 없고, `ROLLBACK`도 불가능합니다.
* **`DROP` (DDL):** 내용물은 물론이고 테이블 구조(식탁) 자체를 박살 내서 완전히 없애버립니다. 역시 `ROLLBACK` 불가능합니다.

### ❌ 함정 2: `LIKE '%'` 의 의미
* **출제 유형:** `SELECT * FROM 회원 WHERE 이름 LIKE '%' ;` 의 결과는?
* **정답 및 해설:** 퍼센트(`%`) 기호는 '모든 문자열'을 의미하므로 테이블의 모든 데이터가 조회될 것 같지만, **NULL 값을 가진 데이터는 조회되지 않습니다!** `LIKE` 연산자도 결국 비교 연산의 일종이므로 모르는 값(NULL)은 매칭할 수 없기 때문입니다.
