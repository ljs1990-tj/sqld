**Q1. 다음 중 데이터 정의어(DDL)에 해당하는 명령어가 아닌 것은?**
1. CREATE
2. ALTER
3. TRUNCATE
4. UPDATE
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. UPDATE는 테이블 안의 데이터를 수정하는 데이터 조작어(DML)입니다.**</span>

**Q2. 다음 중 테이블의 데이터를 삭제하는 명령어에 대한 설명으로 가장 적절하지 않은 것은?**
1. DROP은 테이블의 구조 자체를 완전히 파괴하여 삭제한다.
2. TRUNCATE는 테이블의 구조는 남기고 데이터만 모두 삭제하며, ROLLBACK으로 복구할 수 없다.
3. DELETE는 WHERE 절을 사용하여 특정 데이터만 선택적으로 삭제할 수 있다.
4. DELETE로 데이터를 삭제하면 즉시 데이터베이스에 영구 반영되어 ROLLBACK을 수행할 수 없다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. DELETE는 DML 명령어이므로, COMMIT을 수행하기 전이라면 언제든지 ROLLBACK을 통해 지우기 전 상태로 복구할 수 있습니다.**</span>

**Q3. 직원 테이블에서 이름이 '김'으로 시작하는 직원을 조회하려고 한다. 올바른 WHERE 조건은 무엇인가?**
1. WHERE 이름 LIKE '김_'
2. WHERE 이름 LIKE '김%'
3. WHERE 이름 = '김%'
4. WHERE 이름 IN ('김')
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 퍼센트(%)는 0개 이상의 모든 문자를 의미하므로 '김'으로 시작하는 모든 이름을 찾습니다. 언더바(_)는 딱 1개의 문자만 의미하므로 1번은 '김'으로 시작하는 외자 이름(2글자)만 찾게 됩니다.**</span>

**Q4. 다음 중 아래의 SQL문과 완벽하게 동일한 결과를 출력하는 쿼리는?**
> `SELECT * FROM 사원 WHERE 부서 IN ('영업부', '인사부');`
1. WHERE 부서 = '영업부' AND 부서 = '인사부'
2. WHERE 부서 = '영업부' OR 부서 = '인사부'
3. WHERE 부서 LIKE '영업부' AND 부서 LIKE '인사부'
4. WHERE 부서 BETWEEN '영업부' AND '인사부'
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. IN 연산자는 괄호 안의 값들 중 하나라도 만족하면 조회하는 다중 OR 조건과 완전히 같습니다.**</span>

**Q5. 다음 중 NULL에 대한 설명으로 가장 적절하지 않은 것은?**
1. NULL은 모르는 값(Unknown Value) 또는 아직 정의되지 않은 값을 의미한다.
2. 숫자 10에 NULL을 더하면 계산 결과는 10이 된다.
3. 특정 컬럼의 값이 NULL인지 확인하기 위해서는 'IS NULL' 연산자를 사용해야 한다.
4. 집계 함수(SUM, AVG 등)는 연산 시 NULL 값을 제외하고 계산한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 모르는 값(NULL)에 사칙연산을 수행하면 그 결과는 무조건 NULL이 됩니다. (10 + NULL = NULL)**</span>

**Q6. 다음 SQL 문장의 실행 결과로 알맞은 것은?**
> `SELECT NVL(NULL, 100) + 50 FROM DUAL;`
1. 50
2. 100
3. 150
4. NULL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. NVL(값1, 값2) 함수는 값1이 NULL이면 값2를 반환합니다. 따라서 NVL(NULL, 100)은 100이 되며, 100 + 50 = 150이 출력됩니다.**</span>

**Q7. 다음 중 `SELECT COALESCE(NULL, NULL, 'A', 'B') FROM DUAL;` 의 결과로 알맞은 것은?**
1. NULL
2. A
3. B
4. 에러 발생
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. COALESCE 함수는 괄호 안의 인자들을 순서대로 평가하여 '가장 처음으로 NULL이 아닌 값'을 반환합니다.**</span>

**Q8. 직원의 총 급여를 구하기 위해 기본급(SAL) 컬럼과 보너스(COMM) 컬럼을 더하려고 한다. 이때 보너스가 없는(NULL) 직원의 총 급여가 통째로 NULL이 되어버리는 것을 방지하기 위한 가장 적절한 수식은?**
1. SAL + COMM
2. SAL + NVL(COMM, 0)
3. NVL(SAL + COMM, 0)
4. SAL + ISNULL(COMM, NULL)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 보너스(COMM)가 NULL일 경우 0으로 치환한 뒤 기본급(SAL)과 더해야 안전하게 계산됩니다. 3번처럼 묶어버리면 SAL+COMM 자체가 이미 NULL이 되어버린 후 0으로 바뀌어 기본급마저 날아갑니다.**</span>

**Q9. 사원 테이블에 총 5명의 직원이 있다. 급여(SAL) 컬럼의 값은 각각 [100, 200, NULL, 300, 400] 이다. 이때 `SELECT COUNT(SAL) FROM 사원;` 의 결과는?**
1. 5
2. 4
3. 1000
4. NULL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. COUNT(*)는 전체 행의 수인 5를 반환하지만, COUNT(컬럼명)은 해당 컬럼에서 NULL인 데이터를 제외하고 개수를 셉니다. 따라서 NULL을 제외한 4건이 정답입니다.**</span>

**Q10. DML(데이터 조작어) 명령어를 사용하여 데이터를 추가, 수정, 삭제한 후, 그 변경 사항을 데이터베이스에 영구적으로 확정 및 반영하기 위해 사용하는 TCL 명령어는?**
1. GRANT
2. REVOKE
3. COMMIT
4. ROLLBACK
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. COMMIT을 수행해야만 내가 조작한 데이터가 남들에게도 보이며 DB에 영구 저장됩니다.**</span>
