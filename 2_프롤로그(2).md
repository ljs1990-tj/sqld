# 🎬 [프롤로그 2탄] 데이터베이스의 진짜 마법: 통계와 연결

지금까지는 있는 데이터를 그대로 필터링(WHERE)해서 보는 법을 배웠습니다. 
하지만 대학교 총장님이나 교수님이 원하는 데이터는 "옥한빛 학생의 키" 같은 단순한 정보가 아닙니다. 

"우리 학교 학과별 평균 키는 얼마인가?", "수강신청 내역에 적힌 저 암호 같은 학번 말고, 진짜 학생 이름표를 달아오라!" 같은 요구사항이 쏟아집니다. 이때 필요한 것이 바로 **그룹화(GROUP BY)**와 **조인(JOIN)**입니다.

---

## 📊 실습 1. 통계 내기의 마법 (GROUP BY와 집계 함수)

총장님이 **"우리 학교 각 학과별로 학생 수가 몇 명인지, 평균 키는 얼마나 되는지 당장 보고해!"**라고 지시했습니다. 엑셀이라면 일일이 필터를 걸고 계산기를 두드려야 하지만, DB에서는 단 세 줄이면 끝납니다.

### 1-1. 학과별 통계 내기
```sql
-- 학생들을 '학과(stu_dept)' 기준으로 그룹(팀)을 나눈 뒤, 인원수와 평균 키를 구합니다.
SELECT stu_dept AS 학과명, 
       COUNT(*) AS 학생수, 
       AVG(stu_height) AS 평균키
FROM student
GROUP BY stu_dept;
```
> **💡 여기서 잠깐!**
> 결과를 보면 '전기전자'와 '컴퓨터정보' 학과의 평균 키가 계산되어 나옵니다. 이때, 키가 아직 입력되지 않은(`NULL`) 박희철, 김종헌 학생은 컴퓨터가 알아서 평균 계산에서 제외해 줍니다. 똑똑하죠?

## 🔗 실습 2. 흩어진 퍼즐 맞추기 (JOIN의 필요성)

교수님이 **"이번 학기 수강신청(enrol) 현황을 가져와!"**라고 해서 `enrol` 테이블을 조회해 보았습니다.

### 2-1. JOIN이 없을 때의 비극 (조회해 보세요)
```sql
SELECT * FROM enrol;
```
* **결과 확인:** `101`, `20131001`, `80` ... 이게 도대체 무슨 외계어일까요? 
* **문제점:** 수강신청 테이블에는 데이터 용량을 아끼기 위해 이름 대신 **'과목코드(sub_no)'**와 **'학번(stu_no)'**만 적혀 있습니다. 교수님은 코드가 아니라 **'학생 이름'**을 보고 싶어 합니다.

### 2-2. JOIN의 마법: 테이블 2개 연결하기
이 암호를 풀려면 `enrol` 테이블과 `student` 테이블을 서로 '학번'이라는 공통된 끈으로 묶어주어야 합니다. 이것이 **JOIN(조인)**입니다.

```sql
-- 학생(student) 테이블과 수강(enrol) 테이블을 '학번(stu_no)'을 기준으로 연결합니다!
SELECT student.stu_name AS 학생이름, 
       enrol.sub_no AS 과목코드, 
       enrol.enr_grade AS 점수
FROM student
JOIN enrol ON student.stu_no = enrol.stu_no;
```
* **결과 확인:** 드디어 외계어 같던 학번 자리에 '김종헌', '옥한빛' 같은 진짜 이름이 붙어서 나오기 시작합니다!

### 2-3. (보너스) 끝판왕: 테이블 3개 연결하기
과목코드(`101`, `102`)도 보기 싫습니다! 과목 테이블(`subject`)까지 3개를 통째로 연결해서 완벽한 성적표를 만들어 봅시다.

```sql
-- 학생, 수강, 과목 3개의 테이블을 모두 연결해서 완벽한 성적표를 만듭니다.
SELECT student.stu_name AS 학생이름, 
       subject.sub_name AS 수강과목, 
       enrol.enr_grade AS 점수
FROM enrol
JOIN student ON enrol.stu_no = student.stu_no
JOIN subject ON enrol.sub_no = subject.sub_no;
```

## 🎯 프롤로그를 마치며

어떠신가요? 수백만 건의 데이터라도 `GROUP BY`로 순식간에 통계를 내고, `JOIN`으로 흩어진 데이터를 의미 있는 정보로 엮어내는 것. 이것이 바로 데이터베이스를 배우는 진짜 이유이자, 여러분이 이 강의를 통해 마스터하게 될 핵심 기술입니다.

자, 이제 이 강력한 마법들을 제대로 부리기 위해, 다시 기초 공사(데이터 모델링)부터 탄탄하게 다지러 가볼까요?
