# 🤹 3단계: 테이블 엮어내기 (조인과 그룹화)



개별 테이블에서 데이터를 찾는 법을 알았으니, 이제 여러 테이블을 엮어서 통계를 내고 의미 있는 보고서를 만들어 볼 차례입니다. 이 파트에서는 머릿속으로 **'결과로 나오는 데이터가 몇 줄(Row)일까?'**를 상상하는 것이 가장 중요합니다!

---

## 1. 📊 GROUP BY와 HAVING: "영수증 통계 내기"

수만 건의 결제 내역(데이터)에서 '각 부서별 총 급여'나 '메뉴별 판매량'을 구하고 싶을 때 사용합니다. 데이터를 특정 기준(컬럼)으로 묶어버리는 마법입니다.

* **`GROUP BY` (그룹화):**
  * `SELECT 부서, SUM(급여) FROM 사원 GROUP BY 부서;`
  * **★ SQLD 절대 원칙:** `GROUP BY`를 사용했을 때, `SELECT` 절에는 **그룹화한 기준 컬럼(부서)과 집계 함수(SUM, COUNT 등)**만 적을 수 있습니다. 기준이 아닌 일반 컬럼(예: 사원명)을 적으면 무조건 에러가 납니다!
* **`HAVING` (그룹화 후의 조건):**
  * `WHERE` 절은 데이터를 그룹으로 묶기 **전**에 솎아내는 역할입니다.
  * `HAVING` 절은 데이터를 그룹으로 묶은 **후**에 조건에 맞는 그룹만 남기는 역할입니다. (예: 부서별 총 급여가 5000 이상인 부서만 출력하라 -> `HAVING SUM(급여) >= 5000`)

---

## 2. 🤝 JOIN (조인): "테이블 합체 마법"

시험에서 가장 많은 비중을 차지하며, 수험생들이 가장 많이 틀리는 파트입니다. 두 개 이상의 테이블을 특정 조건(공통된 열)을 기준으로 양옆으로 길게 합칩니다.

* **`INNER JOIN` (내부 조인 - 교집합):** * 양쪽 테이블 모두에 공통으로 존재하는 데이터만 합쳐서 보여줍니다. (가장 흔하게 쓰임)
* **`LEFT / RIGHT OUTER JOIN` (외부 조인):** * 기준이 되는 쪽(LEFT면 왼쪽 테이블)의 데이터는 조건에 안 맞아도 **무조건 100% 다 살려서** 보여주고, 반대쪽 테이블에 없는 값은 그냥 `NULL`로 채워서 합칩니다.
* **`CROSS JOIN` (교차 조인 - 카티시안 곱):**
  * 조인 조건이 없을 때 발생합니다. 왼쪽 테이블 데이터 3건, 오른쪽 4건이면 **3 x 4 = 총 12건**의 모든 경우의 수를 무식하게 다 곱해서 출력합니다.

---

## 3. 🍔 집합 연산자: "결과표를 위아래로 이어 붙이기"

조인(JOIN)이 테이블을 양옆으로 길게 뚱뚱하게 합친다면, **집합 연산자는 쿼리의 결과 2개를 위아래로 길게 이어 붙입니다.** (단, 두 결과의 컬럼 개수와 데이터 타입이 완벽히 똑같아야 합니다.)

* **`UNION` (합집합):** 두 결과표를 위아래로 붙인 뒤, **중복된 행은 딱 한 번만 나오게 제거**하고 싹 정렬해서 보여줍니다. (정렬 작업 때문에 속도가 느립니다.)
* **`UNION ALL` (순수 합집합):** 두 결과표를 중복 검사 없이 그냥 무식하게 위아래로 다 이어 붙입니다. (정렬을 안 해서 **속도가 훨씬 빠릅니다!**)
* **`INTERSECT` (교집합):** 두 결과표에 공통으로 있는 줄만 남깁니다.
* **`MINUS / EXCEPT` (차집합):** 위의 결과표에서 아래의 결과표에 있는 내용을 빼고 남은 것만 보여줍니다.

## 4. 🚨 초보자 단골 함정 (트러블슈팅 가이드)

### ❌ 함정 1: "WHERE 절에서 집계 함수 쓰기"
* **출제 유형:** `SELECT 부서 FROM 사원 WHERE SUM(급여) > 5000 GROUP BY 부서;` (이 쿼리의 문제점은?)
* **정답 및 해설:** `WHERE` 절 안에는 절대 `SUM()`, `COUNT()` 같은 집계 함수를 쓸 수 없습니다! 집계 함수로 조건을 걸고 싶다면 반드시 그룹화가 끝난 뒤인 **`HAVING` 절**에 써야 합니다.

### ❌ 함정 2: JOIN 결과의 '행(Row) 개수' 맞추기
* **출제 유형:** A 테이블에 회원 3명이 있고, B 테이블(주문내역)에 1번 회원의 주문이 2건, 2번 회원의 주문이 1건, 3번 회원의 주문은 없다. 이때 두 테이블을 회원번호로 `INNER JOIN` 하면 총 몇 건이 출력되는가?
* **정답 및 해설:** 정답은 **3건**입니다. `INNER JOIN`은 양쪽에 다 있는 데이터만 남깁니다. 1번 회원은 주문이 2건이므로 두 줄로 뻥튀기되고, 2번 회원은 한 줄, 3번 회원은 주문이 아예 없어서 조인 결과에서 증발해버립니다. (총 2 + 1 = 3건)

### ❌ 함정 3: UNION vs UNION ALL의 특징
* **출제 유형:** 다음 중 조회 속도가 가장 빠른 집합 연산자는?
* **정답 및 해설:** 무조건 **`UNION ALL`**입니다. 출제위원들은 성능을 매우 중요하게 생각하므로, 중복 제거 여부를 묻는 문제에서 `UNION ALL`이 정답인 경우가 아주 많습니다.

## 5. 🧠 배운 내용 확인하기 (미니 퀴즈)

(정답은 드래그하면 보입니다)

**Q1. A 쿼리의 결과가 5건, B 쿼리의 결과가 3건이고, 두 쿼리 결과 사이에 완벽히 중복되는 행이 1건 있다고 가정합시다. 두 쿼리를 `UNION ALL` 로 묶었을 때 최종 출력되는 행(Row)의 개수는 몇 개일까요?**
> 정답: <span style="color:transparent; text-shadow: 0 0 0 #000;">**8건 (UNION ALL은 중복을 제거하지 않고 무조건 다 더하므로 5 + 3 = 8건입니다. 만약 그냥 UNION이었다면 중복 1건이 제거되어 7건이 됩니다.)**</span>

**Q2. 조인(JOIN)을 수행할 때, 양쪽 테이블에 공통된 값이 없더라도 기준이 되는 한쪽 테이블의 데이터는 무조건 누락 없이 모두 출력하게 만드는 조인 방식을 무엇이라고 할까요?**
> 정답: <span style="color:transparent; text-shadow: 0 0 0 #000;">**OUTER JOIN (외부 조인 - LEFT, RIGHT, FULL)**</span>
