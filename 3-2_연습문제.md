**Q1. 데이터 그룹화에 사용되는 `GROUP BY`와 `HAVING` 절에 대한 설명으로 가장 적절하지 않은 것은?**
1. `GROUP BY` 절을 통해 소그룹별 기준을 정한 후, `SELECT` 절에 집계 함수를 사용할 수 있다.
2. 집계 함수를 사용하여 조건을 부여할 때는 `WHERE` 절이 아닌 `HAVING` 절을 사용해야 한다.
3. `HAVING` 절은 통상적으로 `GROUP BY` 절 뒤에 위치한다.
4. `WHERE` 절의 조건으로 먼저 데이터를 걸러낸 후 `GROUP BY`를 수행하는 것은 불가능하며, 무조건 `GROUP BY` 이후에 `HAVING`으로만 걸러내야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 전혀 불가능하지 않습니다! 오히려 WHERE 절을 통해 불필요한 데이터를 먼저 솎아낸 뒤에 GROUP BY를 수행하는 것이 시스템 성능(속도) 면에서 훨씬 유리하며, 실무에서도 적극 권장되는 방식입니다.**</span>

**Q2. 사원 테이블을 '부서명' 기준으로 그룹화하여 부서별 총 급여를 구하려고 한다. 다음 중 문법적으로 올바른 SQL 문장은 무엇인가?**
1. SELECT 부서명, 사원명, SUM(급여) FROM 사원 GROUP BY 부서명;
2. SELECT 부서명, SUM(급여) FROM 사원 GROUP BY 부서명;
3. SELECT 부서명, SUM(급여) FROM 사원 WHERE SUM(급여) >= 5000 GROUP BY 부서명;
4. SELECT SUM(급여) FROM 사원 GROUP BY 부서명, 사원명;
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. GROUP BY 절에 명시된 컬럼(부서명)과 집계 함수(SUM)만 SELECT 절에 올 수 있습니다. 1번은 그룹화 기준이 아닌 '사원명'이 들어가서 에러, 3번은 집계 함수를 WHERE 절에 써서 에러가 발생합니다.**</span>

**Q3. '회원' 테이블(데이터 5건)과 '주문' 테이블(데이터 4건)을 아무런 조인 조건(WHERE 또는 ON) 없이 조인하였다. 이를 교차 조인(CROSS JOIN)이라고 할 때, 출력되는 최종 결과의 행(Row) 수는 몇 개인가?**
1. 4개
2. 5개
3. 9개
4. 20개
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 조인 조건이 없는 카티시안 곱(CROSS JOIN)은 양쪽 테이블의 건수를 곱한(5 x 4 = 20) 모든 경우의 수가 출력됩니다.**</span>

**Q4. A 테이블에는 ID 컬럼에 (1, 2, 3) 값이 있고, B 테이블에는 ID 컬럼에 (1, 2, 2, 4) 값이 있다. 두 테이블을 ID 컬럼을 기준으로 `INNER JOIN` 하였을 때, 출력되는 최종 행(Row) 수는 몇 개인가?**
1. 2개
2. 3개
3. 4개
4. 5개
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. INNER JOIN은 양쪽에 모두 존재하는 값만 출력합니다. ID 1은 한 번 매칭(1건), ID 2는 B 테이블에 두 개가 있으므로 두 번 매칭(2건), ID 3과 4는 한쪽에만 있으므로 버려집니다. 총 1 + 2 = 3건입니다.**</span>

**Q5. 위의 Q4번과 똑같은 A 테이블(1, 2, 3)과 B 테이블(1, 2, 2, 4)을 이번에는 A 테이블을 기준으로 `LEFT OUTER JOIN` 하였다. 출력되는 최종 행(Row) 수는 몇 개인가?**
1. 3개
2. 4개
3. 5개
4. 6개
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. LEFT JOIN은 왼쪽(A)의 데이터는 무조건 다 살립니다. ID 1 매칭(1건), ID 2 매칭(2건), 그리고 ID 3은 매칭되는 값이 B에 없지만 LEFT JOIN이므로 억지로 한 줄을 살리고 B쪽 빈칸은 NULL로 채웁니다. 총 1 + 2 + 1 = 4건입니다.**</span>

**Q6. 다음 중 집합 연산자(Set Operator)를 사용하기 위한 필수 전제 조건으로 가장 올바른 것은?**
1. 두 쿼리의 SELECT 절에 있는 컬럼의 이름이 완벽하게 똑같아야 한다.
2. 두 쿼리의 SELECT 절에 있는 컬럼의 개수와 데이터 타입이 서로 일치해야 한다.
3. 양쪽 쿼리 모두 반드시 GROUP BY 절을 포함하고 있어야 한다.
4. ORDER BY 절은 각각의 쿼리 내부에 개별적으로 작성해야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 집합 연산자(UNION 등)를 쓰려면 컬럼의 '개수'와 '데이터 타입(숫자, 문자 등)'이 같아야 위아래로 이어 붙일 수 있습니다. 컬럼 이름은 달라도 상관없으며, ORDER BY는 무조건 맨 마지막에 딱 한 번만 적어야 합니다.**</span>

**Q7. 다음 집합 연산자 중에서 중복된 데이터를 제거하지 않고 그대로 모두 출력하기 때문에, 정렬 과정이 생략되어 시스템 성능(속도) 면에서 가장 유리한 연산자는 무엇인가?**
1. UNION
2. INTERSECT
3. MINUS
4. UNION ALL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. UNION ALL은 순수하게 위아래로 이어 붙이기만 하므로 중복 제거 연산(정렬)이 발생하지 않아 속도가 매우 빠릅니다.**</span>

**Q8. A 테이블의 조회 결과는 10건, B 테이블의 조회 결과는 7건이다. 두 결과 사이에 완벽히 똑같은 중복 데이터가 3건 있다고 할 때, 두 쿼리를 `UNION` 연산자로 결합하면 출력되는 데이터는 총 몇 건인가?**
1. 10건
2. 14건
3. 17건
4. 20건
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. UNION은 합집합이되 '중복을 제거'합니다. 수학의 합집합 공식 (A + B - 교집합)과 같습니다. 10 + 7 - 3 = 14건이 출력됩니다. (만약 UNION ALL 이었다면 17건입니다.)**</span>

**Q9. 다음 중 '외부 조인(OUTER JOIN)'에 대한 설명으로 가장 적절한 것은?**
1. 조인 조건식의 결과가 참(TRUE)인 행들만 반환한다.
2. 카티시안 곱(Cartesian Product)을 수행한 결과와 동일하다.
3. 기준이 되는 테이블의 데이터는 조인 조건과 상관없이 무조건 결과에 포함된다.
4. 조인에 참여하는 두 테이블의 컬럼 수가 반드시 동일해야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 조인 조건에 맞지 않아도 기준 테이블의 데이터는 모두 살리고, 반대쪽 빈칸은 NULL로 채워넣는 것이 OUTER JOIN의 핵심 특징입니다.**</span>

**Q10. 아래 SQL 문장의 괄호 ( 가 ) 안에 들어갈 알맞은 키워드는?**
> SELECT 부서, AVG(급여) FROM 사원 GROUP BY 부서 ( 가 ) AVG(급여) >= 3000;

1. WHERE
2. ON
3. HAVING
4. IN
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 그룹화(GROUP BY) 이후에 집계 함수(AVG)를 조건으로 걸러내기 위해서는 반드시 HAVING 절을 사용해야 합니다.**</span>
