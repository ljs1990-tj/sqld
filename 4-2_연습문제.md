**Q1. 급여가 높은 순서대로 직원의 순위를 매기려고 한다. 급여가 [500, 400, 400, 300] 일 때, 순위가 [1, 2, 2, 4] 로 출력되게 만드는 윈도우 함수는 무엇인가?**
1. RANK()
2. DENSE_RANK()
3. ROW_NUMBER()
4. NTILE()
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**1번. RANK()는 공동 순위가 존재하면 그다음 순위를 건너뜁니다. 만약 [1, 2, 2, 3]으로 출력되어야 한다면 DENSE_RANK(), [1, 2, 3, 4]로 출력되어야 한다면 ROW_NUMBER()를 써야 합니다.**</span>

**Q2. 다음 중 윈도우 함수(Window Function)에 대한 설명으로 가장 적절하지 않은 것은?**
1. 윈도우 함수는 결과에 대한 데이터 건수(Row 수)를 변형시키지 않고 그대로 유지한다.
2. OVER 절 안에서 PARTITION BY 구문을 사용하여 데이터를 그룹화할 수 있다.
3. 윈도우 함수 내에서 사용하는 집계 함수(SUM, MAX 등)는 기존의 GROUP BY 절과 함께 사용할 수 없다.
4. 윈도우 함수는 일반적인 SELECT 절과 FROM 절 어디에서나 자유롭게 사용할 수 있다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 윈도우 함수는 원칙적으로 SELECT 절과 ORDER BY 절에서만 사용할 수 있습니다. FROM 절이나 WHERE 절에서는 쓸 수 없습니다.**</span>

**Q3. 다음 그룹 함수 중, 괄호 안에 명시된 컬럼들의 결합 가능한 '모든 경우의 수'에 대한 다차원 소계를 생성하는 함수는 무엇인가?**
1. ROLLUP
2. CUBE
3. GROUPING SETS
4. GROUP BY
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. CUBE는 명시된 모든 컬럼의 조합(전체 합계, A 합계, B 합계, A+B 합계)을 산출하므로 시스템 부하가 가장 큰 그룹 함수입니다.**</span>

**Q4. `GROUP BY ROLLUP(부서, 직급)`을 실행했을 때 출력되는 소계/합계의 종류가 아닌 것은?**
1. 부서별, 직급별 소계
2. 부서별 소계
3. 직급별 소계
4. 전체 총계
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. ROLLUP은 왼쪽에서 오른쪽으로 계층 구조를 가집니다. 따라서 '직급별 소계(부서를 무시한 소계)'는 출력되지 않습니다. 직급별 소계가 보고 싶다면 ROLLUP(직급, 부서)로 순서를 바꿔야 합니다.**</span>

**Q5. 다음 중 내가 원하는 특정 그룹의 소계만 선택적으로 골라서 출력하고 싶을 때 사용하는 그룹 함수는 무엇인가? (전체 총계도 자동으로 구해주지 않음)**
1. ROLLUP
2. CUBE
3. GROUPING SETS
4. PIVOT
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. GROUPING SETS는 괄호 안에 적어준 특정 항목들에 대한 독립적인 소계만 계산하여 위아래로 붙여줍니다.**</span>

**Q6. 오라클 계층형 질의(Hierarchical Query)에서 트리의 시작점(루트 노드)을 지정하기 위해 사용하는 키워드는 무엇인가?**
1. CONNECT BY
2. START WITH
3. PRIOR
4. ORDER SIBLINGS BY
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. START WITH 구문을 통해 트리가 어디서부터 시작될지(예: 최상위 사장님) 조건을 지정합니다.**</span>

**Q7. 다음 계층형 질의 문장에서 전개 방향에 대한 설명으로 알맞은 것은?**
> `CONNECT BY PRIOR 사원번호 = 매니저번호`
1. 자식 노드에서 부모 노드 방향으로 전개되는 역방향 전개이다.
2. 부모 노드에서 자식 노드 방향으로 전개되는 순방향 전개이다.
3. 동일한 계층(형제) 간의 정렬을 수행한다.
4. 루트 노드만 조회하고 종료된다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. PRIOR 키워드가 '사원번호(부모)' 쪽에 붙어있습니다. 즉 "방금 읽은 부모의 사원번호가, 다음에 읽을 자식의 매니저번호인 것을 찾아라"는 뜻이므로 사장 -> 사원으로 내려가는 순방향(Top-Down) 전개입니다.**</span>

**Q8. 아래 쿼리의 빈칸 ( 가 ) 에 들어갈 알맞은 윈도우 함수 키워드는?**
> SELECT 이름, 급여, SUM(급여) OVER ( ( 가 ) 부서 ORDER BY 급여 DESC) AS 누적급여 FROM 사원;

1. GROUP BY
2. ORDER BY
3. PARTITION BY
4. GROUPING
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 윈도우 함수 내부(OVER 절)에서 데이터를 그룹으로 나눌 때는 GROUP BY가 아니라 PARTITION BY를 사용해야 합니다.**</span>

**Q9. 윈도우 함수 구문 중 `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` 가 의미하는 것은 무엇인가?**
1. 현재 행부터 맨 마지막 행까지의 범위를 의미한다.
2. 바로 이전 행과 바로 다음 행만 포함하는 범위를 의미한다.
3. 맨 첫 번째 행부터 현재 행까지의 범위를 의미한다. (누계 계산에 사용)
4. 모든 전체 행을 의미한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. UNBOUNDED PRECEDING(맨 처음 행)부터 CURRENT ROW(현재 행)까지를 의미하며, 주로 누적 합계(Running Total)를 구할 때 사용되는 기본 설정값입니다.**</span>

**Q10. 그룹 함수에서 생성된 합계(NULL) 라인인지, 아니면 원래 데이터 자체가 비어있어서 발생한 NULL 라인인지를 구분하기 위해 사용하는 함수는 무엇인가? (합계로 생성된 라인이면 1을, 아니면 0을 반환함)**
1. NVL
2. COALESCE
3. GROUPING
4. NULLIF
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. ROLLUP이나 CUBE를 쓰면 소계 라인에 이름이 없어서 NULL로 표기되는데, 이것이 실제 이름이 없는 사람인지 합계 줄인지 구분할 때 GROUPING() 함수를 씁니다.**</span>
