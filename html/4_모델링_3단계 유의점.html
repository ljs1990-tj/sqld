📖 1과목 핵심 요약: 모델링의 중요성과 3단계 과정

1. ⚠️ 데이터 모델링, 왜 중요할까? (중요성 3가지)

집을 지을 때 뼈대(설계도)가 중요하듯, 정보 시스템 구축에서도 모델링이 성공의 열쇠를 쥡니다.

  • 🎯 파급효과 (Leverage) 수강신청 시스템을 다 완성해서 오픈했는데, 학번 체계를 완전히 바꾸자고 하면 시스템 전체를 갈아엎는 대공사가 일어납니다.
    * 실무 Add-on: 테스트/운영 단계에서 모델을 변경하면 비용이 기하급수적으로 폭발하므로, 초기 설계(분석/설계) 단계부터 신중하게 뼈대를 잡아야 한다는 의미입니다.
  • 🎯 복잡한 요구사항의 간결한 표현 (Conciseness) 복잡한 수강신청 요구사항을 기획자, 개발자, 교직원 누구나 알아볼 수 있게 간결한 그림(ERD)으로 표현해야 합니다. 정보 요구사항이 정확하게 포착되어야 합니다.
  • 🎯 데이터 품질 (Data Quality) 뼈대를 잘 짜둬야 나중에 쓰레기 데이터가 들어오는 것을 막고, 고품질의 데이터를 유지할 수 있습니다. (뒤에 나올 '유의점 3가지'를 피해야 품질이 올라갑니다.)

2. 🚨 데이터 모델링 시 피해야 할 '3대 죄악' (유의점)

시험 문제에 무조건 나오는 단골 파트입니다. 다음 3가지는 시스템을 망치므로 절대 발생하면 안 됩니다.

🚫 중복 (Duplication)

'옥한빛' 학생의 이름과 전화번호가 [학생] 테이블에도 있고 [수강] 테이블에도 똑같이 들어가 있으면 안 됩니다.

📝 기출 포인트: 데이터베이스 내에 똑같은 정보가 여러 곳에 저장되어 공간을 낭비하고 갱신 시 오류를 유발하는 현상. (해결책: 정규화)

🚫 비유연성 (Inflexibility)

대학교에 새로운 학과가 생겼다고 해서 수강신청 시스템 전체를 뜯어고쳐야 한다면 최악의 설계입니다.

📝 기출 포인트: 사소한 업무 변화에 데이터 모델이 수시로 변경되는 현상. (해결책: 데이터의 정의와 데이터의 사용 프로세스를 분리할 것)

🚫 비일관성 (Inconsistency)

[학생] 테이블에는 '휴학' 상태인데, [수강] 테이블에서는 과목을 듣고 있는 '재학' 상태로 나오면 안 됩니다.

📝 기출 포인트: 데이터 간의 상호 연관성이 깨져서 앞뒤가 안 맞는 상태. (해결책: 개발자가 임의로 값을 수정하지 못하게 모델링 단계에서 관계(Relationship)를 명확히 정의할 것)

3. 🧩 데이터 모델링의 3가지 필수 요소 (What, Who, How)

시스템을 이루는 가장 기본적인 재료 3가지입니다. (시험에서는 영어 단어와 매칭해서 나옵니다.)

Things

엔터티 (Entity)
업무가 관여하는 어떤 것
(예: 학생, 과목)

Attributes

속성 (Attribute)
어떤 것이 갖는 성격
(예: 학번, 이름, 성적)

Relationships

관계 (Relationship)
업무가 관여하는 어떤 것 간의 관계
(예: 학생이 과목을 수강한다)

4. 🏗️ 대학교 건물을 짓는 과정: 데이터 모델링 3단계 (초핵심!)

설계도는 한 번에 완성되는 것이 아니라, 큰 그림에서 시작해 점점 구체적인 도면으로 깎여 나갑니다.

1단계
개념적

총장님 보고용 큼직한 스케치 (Conceptual)

가장 추상적이고 포괄적이며 전사적인 모델링입니다. "학생과 과목이 필요하고 둘은 수강 관계로 묶인다"는 식으로 핵심 엔터티만 뽑아서 큼직한 ERD 스케치를 그리는 단계입니다.

🔑 기출 키워드: 추상화 수준 최고, EA(전사적 아키텍처) 수립, 업무 중심
2단계
논리적

★설계자들의 완벽한 청사진 (Logical) - 제일 중요!

가장 핵심이 되는 단계이며 시스템의 뼈대가 완성됩니다. 학생의 식별자(PK: 학번)와 모든 속성을 꼼꼼히 적어내며, 데이터의 중복을 제거하는 '정규화(Normalization)' 작업을 수행합니다.

🔑 기출 키워드: 정규화 수행(필수!), 재사용성(특정 DBMS에 종속되지 않음), 식별자 확정, M:N 관계 해소
3단계
물리적

개발자를 위한 시공 도면 (Physical)

가장 구체적이고 컴퓨터 하드웨어 친화적인 단계입니다. 시스템의 성능, 저장 공간, 디스크 구조를 따져서 실제 오라클(Oracle), MySQL 같은 특정 데이터베이스에 맞춘 테이블 생성(DDL) 직전의 도면을 완성합니다.

🔑 기출 키워드: 구체화 수준 최고, 물리적 성능(인덱스, 파티셔닝, 반정규화) 고려, 특정 DBMS 종속
