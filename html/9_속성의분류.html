📖 1과목 핵심 요약: 속성의 3가지 분류와 도메인

속성(컬럼)도 성격에 따라 여러 가지 기준으로 나눌 수 있습니다.
시험용으로 각 분류의 이름과 예시를 매칭하는 연습을 완벽히 마스터해 봅시다!

🗂️ 1. 특성에 따른 분류 (어떻게 만들어졌는가?)

속성이 태생적으로 원래 있던 것인지, 누군가 인위적으로 만든 것인지 따져봅니다.

  • ① 기본 속성 (Basic Attribute) : "원래부터 있던 진짜 데이터"

    업무 분석을 통해 추출된, 현실 세계에 원래 존재하는 가장 일반적인 속성입니다.

    🎓 수강신청 비유: 학생의 [이름], [주민번호], [생년월일]

  • ② 설계 속성 (Designed Attribute) : "시스템이 만들어낸 꼬리표"

    원래 현실엔 없었지만, 학교에서 학생들을 체계적으로 관리(업무 규칙화)하거나 설계를 용이하게 하기 위해 인위적으로 부여한 코드나 일련번호입니다.

    🎓 수강신청 비유: [학번], [과목코드], [수강신청 접수번호]

🚨 ③ 파생 속성 (Derived Attribute) : "계산된 결과값" (시험 0순위 단골!)

다른 속성의 영향을 받아 계산되어 만들어진 속성입니다.

  • 🎓 비유: 학생의 [총 이수 학점], [평균 평점(GPA)]
  • 💡 실무 & 기출 포인트: 이 값들은 새로운 성적이 입력될 때마다 계속 변하고 다시 계산(동기화)되어야 하므로 데이터의 정합성(일치성)을 유지하기 매우 힘듭니다. 따라서 가급적 적게(최소화) 만들어야 합니다! "파생속성은 많을수록 좋다"는 무조건 오답입니다.

🔑 2. 엔터티 구성방식에 따른 분류 (역할이 무엇인가?)

테이블 안에서 이 컬럼이 어떤 역할을 맡고 있는지(식별자인지 아닌지)에 따른 분류입니다.

① PK (기본키) 속성

수많은 학생 중 특정 학생 1명을 완벽하게 찾아낼 수 있는(식별할 수 있는) 대표 속성입니다.

비유: [학번]

② FK (외래키) 속성

다른 테이블(엔터티)과 관계를 맺기 위해 남의 테이블에서 빌려온 속성입니다.

비유: 수강내역 테이블 안의 [학생 학번]

③ 일반 속성

PK도 아니고 FK도 아닌, 그냥 데이터를 담고 있는 평범한 나머지 속성들입니다.

비유: [이름], [키]

🧩 3. 분해 여부에 따른 분류 (쪼갤 수 있는가?)

  • ① 단일 속성 (Simple Attribute)

    더 이상 의미를 쪼갤 수 없는 독립적인 속성. (예: [나이], [성별])

  • ② 복합 속성 (Composite Attribute)

    여러 개의 의미가 뭉쳐 있어서, 마음만 먹으면 쪼갤 수 있는 속성. (예: [주소] ➔ 시, 구, 동으로 분해 가능)

🚨 ③ 다중값 속성 (Multi-Valued Attribute) : "무조건 분리해라!"

한 칸에 여러 개의 값을 가지는 속성입니다. 앞서 배웠듯 데이터베이스 속성 칸에는 값 1개만 들어가야(원자성) 하므로, 이런 다중값 속성은 반드시 별도의 새로운 하위 엔터티로 분리(1:M 관계)해야 합니다!

🎓 비유: [전화번호] ➔ 집 번호, 핸드폰 번호 2개가 있다면 한 칸에 쉼표(,)로 넣지 말고 '학생_연락처' 테이블을 따로 만들어야 함! (제1정규화 위배 방지)

🏷️ 4. 속성의 명명 규칙 (이름 짓기)

엔터티와 마찬가지로 속성도 이름을 잘 지어야 합니다. 시험에 자주 나오는 O/X 기준입니다.

구분 설명 및 예시
O (권장) 해당 업무(수강신청)에서 사용하는 직관적이고 익숙한 이름을 씁니다.
X (금지) 서술식(문장형) 이름은 절대 금지!
(예: 수강신청을_한_과목의_이름 ➔ 과목명으로 수정)
X (금지) 본인만 아는 약어(줄임말)는 가급적 쓰지 않습니다. (예: 수과명)
O (권장) 전체 데이터 모델에서 유일한 명칭을 쓰는 것이 좋습니다.
(예: 그냥 이름이라고 하기보단 학생이름, 과목이름으로 수식어를 붙여주면 조인(JOIN) 쿼리를 짤 때 충돌/헷갈림을 방지합니다.)

🛡️ 5. 도메인 (Domain)의 완벽 이해 (중요!)

SQLD 시험에서 '도메인'이라는 단어가 나오면 딱 1가지만 떠올리시면 됩니다. 바로 "그 칸(속성)에 들어갈 수 있는 데이터의 허용 범위와 물리적 규칙"입니다. 쓰레기 데이터가 들어오는 것을 막아주는 든든한 '입구 컷(문지기)' 역할을 합니다.

  • ① 타입과 크기: "이 칸은 문자(VARCHAR)만 들어가고, 최대 10글자까지만 허용해!"
  • ② NOT NULL (필수 여부): "이 칸은 비워두면(NULL) 절대 안 돼! 무조건 입력해!"
  • ③ 제약사항 (CHECK 조건): "학생 성별 칸에는 오직 'M' 아니면 'F'만 들어와야 해!"

💻 실무 DB 생성 스크립트(DDL)에 적용된 도메인의 모습

CREATE TABLE 학생 (
  학번 VARCHAR2(10) NOT NULL, -- 도메인: 10자리 문자, 필수
  이름 VARCHAR2(20) NOT NULL,
  성별 CHAR(1) CHECK (성별 IN ('M', 'F')) -- 도메인: M 또는 F만 허용
);
