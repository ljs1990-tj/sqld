# 🗺️ SQLD 완벽 정복을 위한 '거꾸로' 커리큘럼 오리엔테이션

안녕하세요! 데이터베이스의 세계에 오신 것을 환영합니다. 
우리가 앞으로 학습할 순서는 시중에 있는 일반적인 SQLD 수험서나 교재의 목차와는 조금 다릅니다. 왜 1페이지부터 순서대로 배우지 않고 순서를 뒤죽박죽 섞어놓았을까요? 

그 이유는 여러분이 **'포기 없이, 가장 빠르고 직관적으로'** 데이터베이스를 이해하게 만들기 위해서입니다.

---

## 1. 🚨 기존 교재 순서의 치명적인 문제점

대부분의 SQLD 교재는 1과목인 **'데이터 모델링(엔터티, 속성, 정규화 등)'**으로 시작합니다. 
데이터베이스를 한 번도 본 적 없는 사람에게 다짜고짜 "엔터티는 2개 이상의 인스턴스를 가져야 하고, 제3정규화는 이행적 함수 종속을 제거하는 것입니다"라고 가르치면 어떻게 될까요? 

* **결과:** 무슨 외계어인지 전혀 이해하지 못하고, 무작정 암기만 하다가 데이터베이스에 대한 흥미를 잃고 책을 덮어버리게 됩니다. 집을 한 번도 본 적 없는 사람에게 집의 설계도(청사진)부터 들이미는 격입니다.

---

## 2. 💡 우리의 해결책: '거꾸로' 커리큘럼 4단계

우리는 가장 실용적이고 재미있는 것부터 먼저 맛보고, 궁금증이 생길 때쯤 이론을 투입하는 **'영화 예고편(Trailer)'** 방식을 채택했습니다.

### 🎬 프롤로그: "데이터베이스 맛보기" (DML, 기초 SELECT)
* **무엇을 하나요?** 강사가 미리 만들어둔 빈 테이블에 여러분이 직접 데이터를 입력(`INSERT`)해 보고, 그 데이터를 화면에 띄워봅니다(`SELECT`).
* **왜 먼저 하나요?** 눈으로 직접 데이터가 표(엑셀) 형태로 들어가고 나오는 것을 경험하면, "아하! DB라는 게 결국 규칙이 있는 엑셀 같은 거구나!"라는 직관적인 감을 단 10분 만에 잡을 수 있습니다.

### 🕵️ 1단계: "이 테이블들은 왜 쪼개져 있을까?" (1과목: 데이터 모델링)
* **무엇을 하나요?** 앞서 우리가 사용했던 '학생' 테이블과 '성적' 테이블이 왜 하나로 합쳐져 있지 않고 두 개로 쪼개져 있었는지, 그 설계의 비밀(엔터티, 식별자, 정규화)을 파헤칩니다.
* **왜 지금 하나요?** 데이터를 직접 넣어본 뒤에 이론을 배우면, "아, 테이블을 안 쪼개면 데이터를 수정할 때 엄청 귀찮아지는구나!" 하고 실무적인 관점에서 이론을 100% 흡수하게 됩니다. 단순 암기가 아니라 원리를 깨닫는 시간입니다.

### 🏗️ 2단계: "내 손으로 직접 건물 짓고 통제하기" (2과목: DDL, DCL, TCL)
* **무엇을 하나요?** 설계도를 그리는 법을 배웠으니, 이제 실제로 컴퓨터에 테이블(건물)을 생성(`CREATE`)하고, 부수고(`DROP`), 아무나 못 들어오게 보안(`GRANT`)을 걸어보는 관리자 명령어를 배웁니다.
* **왜 지금 하나요?** 설계 이론(1단계)을 실제 물리적인 컴퓨터 환경에 적용해 보는 가장 자연스러운 흐름이기 때문입니다.

### 👨‍🍳 3단계: "데이터 요리 마스터" (2과목: 고급 SELECT, JOIN)
* **무엇을 하나요?** 흩어져 있는 테이블들을 하나로 엮고(`JOIN`), 통계를 내고(`GROUP BY`), 순위를 매기는(`RANK`) 실전 데이터 분석 기술을 익힙니다. SQLD 시험 문제의 70%가 여기서 나옵니다.
* **왜 마지막에 하나요?** 데이터가 어떻게 설계되었고(1과목), 어떻게 만들어졌는지(DDL) 완벽히 이해한 상태에서 쿼리를 짜면, 복잡한 조인(JOIN)이나 서브쿼리도 헷갈리지 않고 정확하게 작성할 수 있기 때문입니다.
