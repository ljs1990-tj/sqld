# 🏆 단기 합격! SQLD (SQL 개발자) 자격증 완벽 공략 로드맵

SQLD 시험은 크게 **1과목(데이터 모델링의 이해, 10문제)**과 **2과목(SQL 기본 및 활용, 40문제)**으로 나뉩니다. 
총점 60점 이상이면 합격이지만, 과목별로 40% 미만을 맞으면 과락으로 불합격 처리되므로 1과목 이론을 절대 무시해서는 안 됩니다. 철저한 기출문제 분석을 바탕으로 구성한 5단계 학습 로드맵입니다.

---

## 🏗️ 1단계: 뼈대 세우기 (1과목: 데이터 모델링의 이해)
직접 쿼리를 치는 것이 아니라, 데이터베이스의 설계도를 그리는 이론 파트입니다. 암기할 내용이 많지만, 나오는 유형이 정해져 있는 전략 과목입니다.

* **핵심 학습 내용:**
  * **엔터티(Entity), 속성(Attribute), 관계(Relationship):** 데이터를 구성하는 3요소의 정의와 표기법
  * **식별자(Identifier):** 기본키(PK)와 외래키(FK)의 개념, 주식별자의 특징
  * **🌟 정규화(Normalization)와 반정규화:** (가장 중요!) 1차, 2차, 3차 정규화의 차이와 성능의 관계, 트랜잭션의 개념
* **합격 포인트:** 개념의 '정의'를 묻는 말장난 문제가 자주 출제됩니다. 단어의 의미를 정확히 짝지어 암기하는 것이 핵심입니다.

## 🏃‍♂️ 2단계: SQL 기본기 다지기 (2과목: SQL 기본)
본격적인 SQL 문법의 시작입니다. 실무에서도 매일 쓰이는 가장 기본적인 쿼리들을 배웁니다.

* **핵심 학습 내용:**
  * **DDL, DML, DCL, TCL:** 명령어들의 종류와 특징 구분 (특히 `COMMIT`과 `ROLLBACK`의 작동 원리)
  * **SELECT와 WHERE:** 조건절 작성법, 부정 연산자, `IN`, `LIKE`, `BETWEEN`
  * **🌟 내장 함수와 NULL 처리:** 문자/숫자/날짜 함수, 그리고 SQLD의 단골손님인 `NVL`, `ISNULL`, `COALESCE`의 차이와 NULL의 연산 규칙
* **합격 포인트:** "숫자 + NULL = ?" 같은 NULL 관련 연산 문제와 형태 변환 함수(`TO_CHAR`, `CAST` 등)의 결과값을 정확히 예측해야 합니다.

## 🤹 3단계: 테이블 엮어내기 (2과목: 조인과 그룹화)
시험의 당락을 좌우하는 2과목의 하이라이트입니다. 테이블이 2개 이상으로 늘어났을 때 데이터가 어떻게 엮이는지 완벽히 이해해야 합니다.

* **핵심 학습 내용:**
  * **GROUP BY와 HAVING:** 그룹화의 원리, 집계 함수(`SUM`, `COUNT`, `MAX` 등) 적용, `WHERE`와의 차이점
  * **🌟 JOIN (조인):** `INNER JOIN`, `LEFT/RIGHT OUTER JOIN`, `CROSS JOIN`
  * **서브쿼리 (Subquery):** 쿼리 안의 쿼리 (스칼라 서브쿼리, 인라인 뷰, 중첩 서브쿼리)
  * **집합 연산자:** `UNION`, `UNION ALL` (중복 제거 여부), `INTERSECT`, `MINUS`
* **합격 포인트:** 조인을 수행했을 때 최종적으로 **'몇 건의 데이터(Row)가 출력되는가?'**를 묻는 함정 문제가 아주 자주 나옵니다. 벤 다이어그램을 그리며 데이터 건수를 세는 연습이 필수입니다.

## ⛰️ 4단계: 마의 구간, 고급 SQL (2과목: SQL 활용)
SQLD 수험생들이 가장 큰 좌절을 겪는 파트입니다. 문법이 복잡하지만, 작동 원리만 이해하면 확실한 점수밭이 됩니다.

* **핵심 학습 내용:**
  * **🌟 윈도우 함수 (Window Function):** `RANK`, `DENSE_RANK`, `ROW_NUMBER`의 순위 산정 방식 차이, `PARTITION BY`
  * **그룹 함수:** `ROLLUP`, `CUBE`, `GROUPING SETS`의 소계/합계 출력 방식 차이
  * **계층형 질의:** `START WITH`, `CONNECT BY`, `PRIOR` (트리 구조 데이터 탐색)
  * **최신 기출 트렌드:** `PIVOT / UNPIVOT` (행과 열 바꾸기), 정규표현식(`REGEXP`)
* **합격 포인트:** 특히 윈도우 함수와 그룹 함수는 눈으로만 보면 절대 풀 수 없습니다. 주어진 표(Table)를 보고 `RANK` 함수가 적용된 결과 테이블을 직접 손으로 그려보는 연습이 필요합니다.

## 🎯 5단계: 실전 모의고사 및 기출문제 풀이 (오답 노트)
SQLD는 문제 은행식 출제 경향이 강하기 때문에 기출문제가 곧 최고의 교재입니다.

* **핵심 학습 내용:**
  * 최신 기출문제(노란책 등) 3~5회분 반복 풀이
  * 자주 틀리는 함정 유형(NULL 집계, JOIN 건수, 정규화) 오답 노트 정리
* **합격 포인트:** 시험장에서 쿼리를 직접 칠 수 없으므로, 눈으로 쿼리를 읽고 머릿속으로 실행 결과를 도출하는 '컴파일러'가 되어야 합니다.

## 🚨 수험생 단골 실수 (트러블슈팅 가이드)

### ❌ 실수 1: COUNT(*)와 COUNT(컬럼명)의 혼동
* **상황:** 집계 함수 문제에서 예상한 출력값과 실제 답이 다릅니다.
* **원인:** `COUNT(*)`는 NULL을 포함한 전체 행(Row)의 개수를 세지만, `COUNT(컬럼명)`은 **해당 컬럼이 NULL인 데이터는 제외하고** 개수를 셉니다. 출제위원들이 가장 좋아하는 함정입니다.

### ❌ 실수 2: 1과목(이론) 암기 미루기
* **상황:** 2과목 SQL은 잘 푸는데, 시험 결과 1과목에서 10문제 중 4문제를 못 맞춰 '과락'으로 불합격합니다.
* **예방책:** 1과목은 양이 적지만 배점이 크고 과락의 위험이 있습니다. 시험 일주일 전부터는 반드시 1과목 이론 요약본을 매일 회독해야 합니다.
