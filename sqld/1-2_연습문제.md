# 📝 SQLD 1과목 실전 연습문제 (1단계: 엔터티, 속성, 관계)

이론의 함정을 피하기 위한 10개의 핵심 기출 변형 문제입니다. 눈으로 직접 풀어보신 뒤 마우스로 드래그하여 정답과 해설을 확인해 보세요!

**Q1. 다음 중 엔터티(Entity)의 특징으로 가장 적절하지 않은 것은?**
1. 반드시 해당 업무에서 필요하고 관리하고자 하는 정보여야 한다.
2. 유일한 식별자에 의해 구분이 가능해야 한다.
3. 하나의 엔터티는 단 하나의 인스턴스(데이터)만 가져야 한다.
4. 엔터티는 반드시 두 개 이상의 속성을 가져야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 하나의 엔터티는 반드시 '두 개 이상'의 인스턴스 집합이어야 합니다. 데이터가 딱 1건뿐이라면 굳이 테이블로 만들 필요가 없습니다.**</span>

**Q2. 다음 중 속성(Attribute)에 대한 설명으로 가장 적절하지 않은 것은?**
1. 업무에서 필요로 하는 데이터 단위 중 더 이상 분리되지 않는 최소의 데이터 단위이다.
2. 하나의 엔터티는 두 개 이상의 속성을 갖는다.
3. 하나의 인스턴스에서 각각의 속성은 상황에 따라 여러 개의 값을 가질 수 있다.
4. 속성도 엔터티와 마찬가지로 일정한 집합의 성격을 가진다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 속성은 반드시 '하나의 값(단일 값)'만을 가져야 합니다. 만약 여러 개의 값을 가진다면 별도의 엔터티로 분리해야 합니다 (제1정규화).**</span>

**Q3. 데이터 모델링 시 "총 결제 금액", "평균 점수"와 같이 다른 속성에 영향을 받아 계산되어 도출되는 속성을 무엇이라고 하는가?**
1. 기본 속성 (Basic Attribute)
2. 설계 속성 (Designed Attribute)
3. 파생 속성 (Derived Attribute)
4. PK 속성 (Primary Key Attribute)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 파생 속성입니다. 데이터의 무결성을 유지하기 어렵기 때문에 가급적 적게 정의하는 것이 좋습니다.**</span>

**Q4. 엔터티 간의 관계(Relationship)를 표기할 때 포함되어야 하는 3가지 핵심 요소가 아닌 것은?**
1. 관계명 (Relationship Name)
2. 관계차수 (Relationship Degree)
3. 관계선택성 (Relationship Optionality)
4. 관계식별자 (Relationship Identifier)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 관계의 3요소는 관계명(이름), 관계차수(1:1, 1:M 등), 관계선택성(필수/선택)입니다.**</span>

**Q5. 다음 중 엔터티의 이름을 짓는 명명 규칙으로 가장 적절하지 않은 것은?**
1. 현업 업무에서 실제로 사용하는 용어를 사용한다.
2. 시스템 구조 파악을 위해 가능하면 약어를 적극적으로 사용한다.
3. 복수형이 아닌 단수 명사를 사용한다.
4. 모든 엔터티에서 유일하게 이름이 부여되어야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 엔터티 이름은 누구나 직관적으로 이해할 수 있도록 가능하면 약어 사용을 자제해야 합니다.**</span>

**Q6. 다음에서 설명하는 엔터티의 종류는 무엇인가?**
> "업무를 수행함에 따라 발생되며, 데이터 발생량이 가장 많고 각종 통계자료에 주로 사용된다. (예: 주문, 청구, 미수)"
1. 유형 엔터티 (Tangible Entity)
2. 개념 엔터티 (Conceptual Entity)
3. 사건 엔터티 (Event Entity)
4. 중심 엔터티 (Main Entity)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 데이터가 지속적으로 발생하고 통계에 주로 쓰이는 것은 '사건 엔터티'입니다.**</span>

**Q7. 특정 대학교의 '수강신청' 업무 시스템을 구축하려고 한다. 다음 중 이 시스템의 엔터티로 도출하기에 가장 부적절한 것은?**
1. 학생
2. 교수
3. 개설과목
4. 교내 식당 식자재
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. '교내 식당 식자재'는 명사이고 데이터 집합일 수 있으나, '수강신청'이라는 해당 업무와는 전혀 관련이 없으므로 엔터티가 될 수 없습니다.**</span>

**Q8. 다음 중 '설계 속성(Designed Attribute)'에 해당하는 가장 적절한 예시는 무엇인가?**
1. 회원의 본명
2. 상품의 제조 원가
3. 전체 주문의 총 결제 금액
4. 시스템에서 고유하게 부여한 회원번호(User ID)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 기본 속성(이름, 원가) 외에 시스템의 규칙화 및 식별을 위해 새롭게 설계하여 만들어낸 속성(코드, 번호 등)이 설계 속성입니다.**</span>

**Q9. '부서'와 '사원' 엔터티가 있다. "모든 사원은 반드시 하나의 부서에 소속되어야 하지만, 신설된 부서에는 아직 사원이 한 명도 없을 수도 있다." 이 상황에서 '부서' 관점(부서 -> 사원)에서의 관계선택성은 무엇인가?**
1. 필수(Mandatory) 참여
2. 선택(Optional) 참여
3. 다대다(M:N) 참여
4. 1:1 참여
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 부서 입장에서는 사원이 '없을 수도' 있으므로 선택적 참여가 됩니다. 반대로 사원 입장에서는 부서가 반드시 있어야 하므로 필수 참여입니다.**</span>

**Q10. 다음은 데이터 모델링의 핵심 용어에 대한 설명이다. ( 가 ) 와 ( 나 ) 에 들어갈 알맞은 단어는?**
> "하나의 ( 가 ) 는 여러 개의 ( 나 ) 집합으로 구성된다. 예를 들어 '고객'이라는 ( 가 ) 안에 '홍길동', '이순신' 등의 ( 나 ) 가 존재한다."
1. (가) 속성, (나) 엔터티
2. (가) 엔터티, (나) 인스턴스
3. (가) 인스턴스, (나) 속성
4. (가) 관계, (나) 인스턴스
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 테이블 자체(고객)를 엔터티라고 부르며, 그 안에 들어가는 실제 데이터 한 줄 한 줄(홍길동, 이순신)을 인스턴스라고 부릅니다.**</span>
