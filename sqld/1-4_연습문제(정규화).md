# 📝 SQLD 1과목 실전 연습문제 (1.5단계: 정규화와 반정규화)

---

**Q1. 정규화를 수행하지 않은 데이터베이스에서 데이터의 중복으로 인해 발생하는 '이상 현상(Anomaly)'의 종류가 아닌 것은?**
1. 삽입 이상 (Insert Anomaly)
2. 삭제 이상 (Delete Anomaly)
3. 갱신 이상 (Update Anomaly)
4. 검색 이상 (Search Anomaly)

**Q2. 다음 중 제1정규화(1NF)를 수행해야 하는 상황에 대한 설명으로 가장 알맞은 것은?**
1. 테이블의 기본키(PK)가 아닌 일반 속성들 간에 함수적 종속성이 존재할 때
2. 테이블의 기본키가 2개 이상의 속성으로 이루어진 복합키일 때, 일부 속성에만 종속되는 일반 속성이 존재할 때
3. 테이블의 특정 컬럼 안에 여러 개의 값(다중 값)이 콤마(,) 등으로 나열되어 들어가 있을 때
4. 조회 성능을 향상시키기 위해 의도적으로 중복 데이터를 허용할 때


**Q3. '주문상세' 테이블의 기본키(PK)는 [주문번호 + 상품코드]로 구성되어 있다. 그런데 이 테이블 안에 일반 속성으로 '상품명'과 '상품단가'가 포함되어 있다. '상품명'과 '상품단가'는 오직 '상품코드'만 알면 결정되는 값이다. 이 테이블을 정규화하기 위해 가장 알맞은 조치는?**
1. 제1정규화를 수행하여 상품명과 상품단가를 다중 값으로 만든다.
2. 제2정규화를 수행하여 [상품코드, 상품명, 상품단가]를 새로운 '상품' 테이블로 분리한다.
3. 제3정규화를 수행하여 주문번호와 상품코드를 분리한다.
4. 반정규화를 수행하여 상품명 컬럼을 삭제한다.


**Q4. 고객 테이블의 기본키(PK)는 '고객번호'이다. 테이블 안에는 일반 속성으로 [회원등급]과 [등급별_할인율]이 있다. 그런데 '등급별_할인율'은 고객번호가 아니라 '회원등급'에 의해 결정되고 있다. (고객번호 ➔ 회원등급 ➔ 등급별_할인율) 이러한 '이행적 함수 종속'을 제거하기 위해 수행해야 하는 정규화는?**
1. 제1정규화 (1NF)
2. 제2정규화 (2NF)
3. 제3정규화 (3NF)
4. BCNF


**Q5. 아직 상품을 한 번도 주문하지 않은 '신규 고객'을 데이터베이스에 등록하려고 한다. 그런데 고객 정보와 주문 정보가 하나로 합쳐진 테이블의 기본키가 (주문번호)로 설정되어 있어서, 가짜 주문번호를 입력하지 않으면 신규 고객 자체를 등록할 수 없는 현상이 발생했다. 이는 어떤 이상 현상에 해당하는가?**
1. 갱신 이상
2. 삭제 이상
3. 삽입 이상
4. 무결성 이상


**Q6. 다음 중 '반정규화(Denormalization)'에 대한 설명으로 가장 적절하지 않은 것은?**
1. 반정규화는 데이터의 무결성을 보장하기 위해 테이블을 더욱 잘게 분할하는 과정이다.
2. 시스템의 조회 성능을 향상시키기 위해 의도적으로 데이터의 중복을 허용하는 기법이다.
3. 반정규화를 과도하게 적용하면 데이터를 수정(Update)할 때 오히려 성능이 저하되고 데이터 불일치가 발생할 수 있다.
4. 디스크의 I/O를 줄이고 복잡한 조인(JOIN) 경로를 단축시키기 위해 수행한다.

**Q7. 다음 중 정규화(Normalization)의 주된 목적으로 가장 거리가 먼 것은?**
1. 데이터 구조의 안정성 및 무결성 유지
2. 데이터의 중복 최소화
3. 이상 현상(Anomaly)의 방지
4. 복잡한 다중 테이블 조인(JOIN)을 유도하여 데이터 조회 속도를 극대화

**Q8. 반정규화 기법 중 하나인 '테이블 병합'을 수행하기에 가장 적절한 상황은 언제인가?**
1. 한 테이블에 속성(컬럼)이 너무 많아 디스크의 여러 블록에 걸쳐 저장되어 I/O가 많이 발생할 때
2. 두 테이블이 항상 1:1 또는 1:M 관계로 조인되어 함께 조회되는 빈도가 압도적으로 높을 때
3. 특정 컬럼의 값을 계산하는 로직이 너무 단순하여 언제든 빠르게 계산할 수 있을 때
4. 데이터의 갱신(Update)이 하루에도 수십만 건씩 빈번하게 일어날 때


**Q9. '학생' 테이블에 "전체 총 결제 금액"이라는 파생 속성(계산된 값) 컬럼을 추가하려고 한다. 이러한 컬럼 중복 반정규화를 수행했을 때 발생할 수 있는 가장 치명적인 단점은?**
1. 데이터를 조회(SELECT)하는 속도가 현저히 느려진다.
2. 조인 연산이 늘어나서 쿼리가 복잡해진다.
3. 누군가 과거의 결제 금액 데이터를 수정(Update)할 경우, 파생 속성의 값과 실제 결제 내역 간의 데이터 불일치(무결성 훼손)가 발생할 수 있다.
4. 데이터베이스의 저장 공간이 절약된다.

**Q10. "A ➔ B" 형태의 함수적 종속성에서 A를 부르는 명칭과 B를 부르는 명칭으로 올바르게 짝지어진 것은?**
1. A: 종속자, B: 결정자
2. A: 결정자, B: 종속자
3. A: 다중값, B: 원자값
4. A: 외래키, B: 기본키
