## [1과목] 데이터 모델링의 이해 (Q1 ~ Q4)

**Q1. 다음 중 데이터 모델링의 핵심 요소인 엔터티(Entity)에 대한 설명으로 가장 적절하지 않은 것은?**
1. 엔터티는 사람, 장소, 물건, 사건 등과 같이 명사로 표현할 수 있는 객체이다.
2. 시스템 구축 시, 해당 업무에서 필요로 하고 관리해야 하는 정보여야만 엔터티가 될 수 있다.
3. 데이터베이스 성능 향상을 위해 하나의 엔터티는 단 하나의 인스턴스(데이터)만 갖도록 설계하는 것이 원칙이다.
4. 엔터티는 유일한 식별자(Identifier)에 의해 식별이 가능해야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 하나의 엔터티는 반드시 '두 개 이상'의 인스턴스(데이터) 집합이어야 합니다. 데이터가 1건뿐이라면 굳이 테이블을 만들 필요가 없습니다.**</span>

**Q2. 다음 중 기본 속성, 설계 속성, 파생 속성에 대한 설명으로 올바르게 짝지어진 것은?**
1. 기본 속성: 예금잔고, 총 주문금액
2. 설계 속성: 상품코드, 지점번호
3. 파생 속성: 고객의 본명, 생년월일
4. 파생 속성: 업무상 새롭게 부여된 일련번호
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 1번은 파생 속성, 3번은 기본 속성, 4번은 설계 속성에 대한 설명입니다. 상품코드처럼 시스템 식별을 위해 인위적으로 만든 것이 설계 속성입니다.**</span>

**Q3. 다음은 어떤 정규화(Normalization) 단계에 대한 설명인가?**
> "테이블의 기본키(PK)가 복합키(2개 이상의 속성)로 구성되어 있을 때, 기본키 전체가 아닌 기본키의 '일부'에만 종속되는 일반 속성들을 분리하여 새로운 테이블을 만드는 과정이다. (부분 함수 종속 제거)"
1. 제1정규화
2. 제2정규화
3. 제3정규화
4. BCNF
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 복합키의 횡포에서 벗어나 일반 컬럼들을 독립시키는 '부분 함수 종속 제거'는 제2정규화입니다.**</span>

**Q4. 시스템의 조회 성능을 향상시키기 위해 의도적으로 정규화 원칙을 위배하고 데이터의 중복을 허용하는 '반정규화(Denormalization)'의 기법으로 가장 적절하지 않은 것은?**
1. 1:1 관계로 자주 조인되는 두 테이블을 하나로 병합한다.
2. 데이터를 빠르게 조회하기 위해 파생 속성(계산된 총합계 등) 컬럼을 추가한다.
3. 데이터의 일관성과 무결성을 극대화하기 위해 테이블을 속성 단위로 잘게 쪼갠다.
4. 본사 테이블과 지사 테이블처럼 지역별로 데이터를 분산시키기 위해 테이블을 수평 분할한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 무결성을 극대화하기 위해 테이블을 잘게 쪼개는 것은 반정규화가 아니라 '정규화'의 목적입니다.**</span>

## [2과목] SQL 기본 및 활용 (Q5 ~ Q20)

**Q5. 다음 중 명령어의 분류가 올바르게 연결되지 않은 것은?**
1. DDL : CREATE, ALTER, DROP
2. DML : INSERT, UPDATE, DELETE
3. TCL : COMMIT, ROLLBACK
4. DCL : SELECT, TRUNCATE
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. SELECT는 DML(데이터 조작어)이며, TRUNCATE는 DDL(데이터 정의어)입니다. DCL(데이터 제어어)은 GRANT, REVOKE 입니다.**</span>

**Q6. 다음 SQL 문장의 실행 결과로 알맞은 것은?**
> `SELECT 100 + NULL FROM DUAL;`
1. 100
2. 0
3. 에러 발생
4. NULL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 모르는 값(NULL)에 사칙연산을 수행하면 결과는 무조건 NULL이 전염됩니다.**</span>

**Q7. 다음 중 문자열 조회 조건 `WHERE 이름 LIKE '_길%'` 이 의미하는 것으로 알맞은 것은?**
1. 이름이 '길'로 시작하는 모든 사람
2. 이름에 '길'이 포함된 모든 사람
3. 이름의 두 번째 글자가 '길'인 모든 사람
4. 이름이 2글자이면서 '길'로 끝나는 사람
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 언더바(_)는 무조건 글자 1개를 의미하고, 퍼센트(%)는 0개 이상의 모든 문자를 의미합니다. 따라서 첫 글자는 아무거나 1글자가 오고, 두 번째가 '길'이며, 그 뒤로는 몇 글자가 오든 상관없는 이름을 찾습니다.**</span>

**Q8. 사원 테이블에서 보너스(COMM)가 지급되지 않은(NULL) 직원을 조회하기 위한 가장 올바른 조건식은?**
1. WHERE COMM = NULL
2. WHERE COMM IS NULL
3. WHERE NVL(COMM, 0) = NULL
4. WHERE COMM IN (NULL)
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. NULL은 비교 연산자(=, >, <)로 찾을 수 없으며 오직 IS NULL, IS NOT NULL 연산자만 사용해야 합니다.**</span>

**Q9. [상품] 테이블에 총 10건의 데이터가 있다. 그중 '할인율' 컬럼의 값이 입력된 데이터는 7건이고, 3건은 NULL이다. 이때 `SELECT COUNT(할인율) FROM 상품;` 의 결과는?**
1. 10
2. 7
3. 3
4. 0
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. COUNT(*)는 모든 행을 세지만, COUNT(컬럼명)은 해당 컬럼에서 NULL인 데이터를 제외하고 개수를 셉니다.**</span>

**Q10. `SELECT COALESCE(NULL, NULL, '사과', '바나나') FROM DUAL;` 의 출력 결과로 알맞은 것은?**
1. NULL
2. 사과
3. 바나나
4. 사과, 바나나
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. COALESCE 함수는 나열된 인자들을 순서대로 확인하여 '가장 먼저 나오는 NULL이 아닌 값'을 반환합니다.**</span>

**Q11. 다음 중 SQL의 논리적 실행 순서로 올바른 것은?**
1. SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY
2. FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
3. FROM → SELECT → WHERE → GROUP BY → HAVING → ORDER BY
4. FROM → WHERE → SELECT → GROUP BY → HAVING → ORDER BY
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 컴퓨터는 테이블을 먼저 찾고(FROM), 조건에 맞는 데이터를 추려낸 뒤(WHERE), 묶고(GROUP BY), 묶은 결과를 필터링하고(HAVING), 화면에 보여줄 컬럼을 선택한 후(SELECT), 마지막에 정렬(ORDER BY)합니다.**</span>

**Q12. 조인(JOIN)에 대한 설명으로 가장 적절하지 않은 것은?**
1. INNER JOIN은 조인 조건에 일치하는, 양쪽 테이블 모두에 존재하는 데이터만 반환한다.
2. LEFT OUTER JOIN은 조인 조건에 일치하지 않아도 왼쪽(기준) 테이블의 데이터는 무조건 결과에 포함시킨다.
3. CROSS JOIN은 조인 조건이 없는 경우 발생하며, 두 테이블 행의 개수를 곱한 것만큼의 결과가 나온다.
4. 조인에 참여하는 두 테이블의 컬럼 개수는 반드시 동일해야 한다.
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 집합 연산자(UNION)는 컬럼 개수가 동일해야 하지만, 조인(JOIN)은 조인 조건(보통 PK와 FK)만 일치한다면 컬럼 개수와는 전혀 무관하게 테이블을 옆으로 이어 붙일 수 있습니다.**</span>

**Q13. '학생(A)' 테이블에는 데이터가 4건, '수강과목(B)' 테이블에는 데이터가 3건 있다. 두 테이블을 아무런 조건 없이 CROSS JOIN 하였을 때 출력되는 행(Row)의 수는?**
1. 3건
2. 4건
3. 7건
4. 12건
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. 카티시안 곱(CROSS JOIN)은 양쪽 테이블의 건수를 무식하게 곱합니다. 4 x 3 = 12건입니다.**</span>

**Q14. 다음 중 쿼리 수행 속도(성능)가 가장 빠른 집합 연산자는 무엇인가?**
1. UNION
2. INTERSECT
3. MINUS
4. UNION ALL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**4번. UNION ALL은 중복 제거를 위한 정렬 작업(Sort)을 수행하지 않고 데이터를 있는 그대로 위아래로 이어 붙이기 때문에 가장 빠릅니다.**</span>

**Q15. 메인 쿼리의 WHERE 절에 사용되는 서브쿼리(Subquery) 중, 서브쿼리의 결과가 여러 행(Multi-row)으로 반환될 때 사용할 수 **없는** 연산자는 무엇인가?**
1. IN
2. ANY
3. =
4. ALL
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 등호(=)나 부등호(>, <) 같은 단일 행 연산자는 서브쿼리의 결과가 딱 1건일 때만 쓸 수 있습니다. 여러 행이 나오면 "누구랑 같다는 거야?" 하고 에러가 납니다.**</span>

**Q16. 급여(SAL)가 [4000, 3000, 3000, 2000] 인 4명의 직원이 있다. 윈도우 함수를 사용하여 급여 순위를 [1, 2, 2, 3] 형태로 출력하고자 할 때 빈칸에 들어갈 함수는 무엇인가?**
> SELECT 이름, 급여, (   빈칸   ) OVER(ORDER BY 급여 DESC) AS 순위 FROM 사원;
1. RANK()
2. DENSE_RANK()
3. ROW_NUMBER()
4. NTILE()
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**2번. 공동 순위가 존재할 때 그다음 순위를 건너뛰지 않고 촘촘하게(Dense) 매기는 함수는 DENSE_RANK() 입니다.**</span>

**Q17. 윈도우 함수의 OVER 절 내부에서 데이터를 특정 기준(예: 부서)으로 소그룹화하기 위해 사용하는 구문은 무엇인가?**
1. GROUP BY 부서
2. ORDER BY 부서
3. PARTITION BY 부서
4. DIVIDE BY 부서
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. 윈도우 함수 내에서는 GROUP BY가 아니라 PARTITION BY를 사용하여 파티션(그룹)을 나눕니다.**</span>

**Q18. 아래 쿼리의 결과로 출력되지 **않는** 소계/총계는 무엇인가?**
> SELECT 부서, 직급, SUM(급여) FROM 사원 GROUP BY ROLLUP(부서, 직급);
1. 부서별, 직급별 소계

> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. ROLLUP은 나열된 컬럼에 대해 왼쪽에서 오른쪽으로 계층적인 합계를 구합니다. 따라서 부서를 무시한 '직급별 단독 소계'는 출력되지 않습니다.**</span>

**Q19. 오라클의 계층형 질의문에서 순방향(부모 ➔ 자식)으로 트리를 탐색하고자 할 때, PRIOR 키워드의 올바른 위치는?**
> (가정: 부모 컬럼은 '상위부서코드', 자식 컬럼은 '부서코드'이다)
1. CONNECT BY PRIOR 상위부서코드 = 부서코드
2. CONNECT BY 상위부서코드 = PRIOR 부서코드

> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**1번. "방금 읽은 부모의 [상위부서코드]가 다음에 읽을 자식의 [부서코드]인 사람을 찾아라"는 뜻이므로, 순방향 전개를 하려면 PRIOR 키워드를 부모 쪽 컬럼에 붙여야 합니다.**</span>

**Q20. 테이블의 모든 데이터를 삭제하려고 한다. 이 중 DDL 명령어에 해당하여 ROLLBACK으로 복구가 불가능하며, 테이블의 껍데기(구조)는 남겨두고 데이터만 빠르게 비우는 명령어는 무엇인가?**

2. DELETE TABLE
3. TRUNCATE TABLE
4. FORMAT TABLE
> 정답 및 해설: <span style="color:transparent; text-shadow: 0 0 0 #000;">**3번. DROP은 테이블 자체를 파괴하고, DELETE는 복구가 가능한 DML입니다. 껍데기만 남기고 영구적으로 비우는 DDL 명령어는 TRUNCATE 입니다.**</span>
