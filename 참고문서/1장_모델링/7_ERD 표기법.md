# 📖 1과목 핵심 요약: ERD 표기법과 작성 6단계 절차


## 2. 🗺️ 대학교 설계도 그리기: ERD 작성 6단계 (초핵심!)
건축가가 설계도를 그릴 때도 순서가 있듯, ERD를 그리는 완벽한 6단계 절차가 있습니다. **시험에서 "다음 중 ERD 작성 순서로 올바른 것은?" 하고 순서 맞추기 문제로 무조건 출제됩니다.**

* **기억할 순서:** **도출 ➔ 배치 ➔ 관계 설정 ➔ 관계명 ➔ 참여도 ➔ 필수 여부**

### 🟢 1단계: 엔터티를 그린다 (도출)
* **설명:** 시스템에 필요한 명사(주인공)들을 찾아 네모 박스로 그립니다.
* **수강신청 비유:** 백지에 **[학생]**, **[수강내역]**, **[과목]** 이라는 3개의 네모 박스를 덩그러니 그립니다.

### 🟢 2단계: 엔터티를 적절하게 배치한다
* **설명:** 아무렇게나 두는 것이 아니라, 가장 중요하고 기준이 되는 엔터티를 **왼쪽 상단(Top-Left)**에 두고, 연결 고리가 되는 중심 엔터티를 **중앙**에 배치합니다.
* **수강신청 비유:** 대학교의 가장 기본인 **[학생]** 박스를 왼쪽 위에 둡니다. 그리고 학생과 과목을 이어주는 핵심인 **[수강내역]** 박스를 한가운데에 배치합니다.

### 🟢 3단계: 엔터티 간의 관계를 설정한다
* **설명:** 네모 박스들 사이에 선을 그어서 서로 어떤 연관이 있는지 이어줍니다. (주의: 빙글빙글 도는 순환/Circle 관계가 발생하지 않도록 조심해야 합니다.)
* **수강신청 비유:** [학생]과 [수강내역] 사이에 선을 긋고, [과목]과 [수강내역] 사이에도 선을 긋습니다.

### 🟢 4단계: 관계명을 기술한다
* **설명:** 선만 그어두면 무슨 뜻인지 모르니, 선 위에 동사(현재형)로 이름을 적어줍니다. 너무 포괄적인 단어는 피합니다. (단, 실무에서는 너무 뻔하면 생략하기도 합니다.)
* **수강신청 비유:** * 학생 ➔ 수강내역 선 위에: **"신청한다"**
  * 과목 ➔ 수강내역 선 위에: **"포함된다"**

### 🟢 5단계: 관계의 참여도를 기술한다 (까마귀 발 그리기)
* **설명:** 1:1인지, 1:M(다수)인지, M:N인지 개수(수량)를 표시합니다.
* **수강신청 비유:** 학생 1명이 '여러 개'의 과목을 수강할 수 있으므로, 수강내역 쪽에 **까마귀 발(삼지창)** 모양을 그려줍니다. (1:M 관계)

### 🟢 6단계: 관계의 필수 여부를 기술한다 (동그라미 그리기)
* **설명:** 이 관계가 반드시 있어야만 하는가(필수), 없어도 되는가(선택)를 표시합니다. 선택 참여일 경우 선 끝에 **동그라미(O)**를 그립니다.
* **수강신청 비유:** 신입생은 아직 수강신청을 '0건' 했을 수도 있습니다. 수강내역이 없을 수도 있으므로, 선 끝에 **선택(O) 기호**를 그려줍니다.

# 📖 1과목 핵심 요약 : ERD 관계 선 그리기 (IE 표기법)

앞서 ERD 작성 6단계 중 5단계(참여도)와 6단계(필수 여부)를 시각적으로 어떻게 그리는지 보여주는 파트입니다. 시험에 나오는 기호는 딱 3가지만 기억하시면 됩니다.

## 1. 🔱 관계의 참여도 (1인가, 여러 개/M인가?)
선 끝이 갈라지는지 아닌지로 개수를 파악합니다.

* **작대기 하나 ( `|` ) : 1개**
  * **비유:** 수강내역 1줄은 반드시 **'1명'**의 학생에게만 속합니다. (수강내역 쪽에서 학생 쪽을 바라볼 때)
* **까마귀 발 / 삼지창 ( `∈` ) : 여러 개 (M, 다수)**
  * **비유:** 학생 1명은 **'여러 개'**의 수강내역을 가질 수 있습니다. (학생 쪽에서 수강내역 쪽을 바라볼 때)

## 2. ⭕ 관계의 필수 여부 (반드시 있어야 하는가?)
선 끝에 동그라미가 있는지, 작대기가 있는지를 봅니다.

* **작대기 ( `|` ) : 필수 (Mandatory)**
  * **비유:** 수강내역이 존재하려면 **반드시** 그 주인이 되는 '학생'이 있어야 합니다. 유령 수강내역은 불가능하므로 학생 쪽 선 끝에는 작대기(`|`)를 긋습니다.
* **동그라미 ( `O` ) : 선택 (Optional)**
  * **비유:** 갓 입학한 신입생은 아직 수강신청을 안 했을 수도 있습니다. 즉, 학생 입장에서 수강내역은 '없을 수도(0개)' 있으므로 수강내역 쪽 선 끝에는 동그라미(`O`)를 긋습니다.

> **💡 요약 조합해 보기 (1:M 선택 관계)**
> 학생 ───|초───────O∈ 수강내역
> *(해석: 학생 1명은 수강내역을 안 가질 수도(O) 여러 개 가질 수도(∈) 있다. 수강내역은 반드시(|) 1명(|)의 학생을 가져야 한다.)*
