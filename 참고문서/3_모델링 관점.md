# 👁️ 1과목 심화: 데이터 모델링의 3가지 관점 (학생 테이블 비유)

시스템을 구축할 때, 기획자와 개발자들은 세 개의 다른 안경을 번갈아 끼며 설계도를 검토합니다. SQLD 시험에서는 각 관점이 **'무엇(What)'**을 묻는지, **'어떻게(How)'**를 묻는지를 구분하는 문제가 자주 출제됩니다.

---

## 1. 🗃️ 데이터 관점 (Data Perspective) : "명사(Noun)의 관점, What"
* **핵심 질문:** 이 대학교 업무를 하려면 **'어떤 데이터(What)'**가 필요하고, 그 데이터들끼리는 **'어떤 관계'**가 있는가?
* **학생 테이블 비유:** * "수강신청을 하려면 일단 **[학생]** 정보가 있어야 하고, **[과목]** 정보가 있어야겠네."
  * "학생은 여러 과목을 들을 수 있고, 과목도 여러 학생을 받을 수 있으니 **[수강내역]**이라는 데이터로 연결(관계)해 줘야겠다."
* **특징:** 행동이나 움직임은 전혀 신경 쓰지 않고, 오직 멈춰있는 데이터의 구조(엔터티, 속성, 관계)에만 집중하는 관점입니다.

## 2. ⚙️ 프로세스 관점 (Process Perspective) : "동사(Verb)의 관점, How"
* **핵심 질문:** 이 대학교에서는 실제로 **'무슨 일(업무)'**이 일어나고 있고, 그 일은 **'어떻게(How)'** 처리되는가?
* **학생 테이블 비유:**
  * "수강신청 기간이 되면, 학생이 로그인해서 ➔ 과목을 검색하고 ➔ 수강 버튼을 누르고 ➔ 정원이 다 찼는지 검사하고 ➔ 신청을 완료한다."
* **특징:** 데이터가 어떻게 생겼는지는 관심 없고, 오직 일의 순서, 흐름, 로직(Logic) 등 움직이는 '과정'에만 집중하는 관점입니다.

## 3. 🔄 데이터와 프로세스의 상관 관점 (Interaction Perspective) : "CRUD"
* **핵심 질문:** 저 **'프로세스(일)'**가 일어날 때, 우리의 **'데이터'**는 어떻게 변하는가?
* **학생 테이블 비유:**
  * "학생이 수강 버튼을 누르는 **일(프로세스)**이 발생하면, `enrol(수강내역)` 테이블에 새로운 데이터 한 줄이 **추가(INSERT)**되어야 해!"
  * "교수님이 성적을 매기는 **일(프로세스)**을 하면, `enrol` 테이블의 빈칸이었던 점수가 **수정(UPDATE)**되어야 해!"
* **특징:** 1번(데이터)과 2번(프로세스)을 서로 교차 검증하는 단계입니다. 실무에서는 이를 **CRUD 매트릭스** (Create, Read, Update, Delete)라는 표를 그려서, 업무에 의해 데이터가 정상적으로 변화하는지 꼼꼼하게 따져봅니다.

---

## 🚨 SQLD 실전 기출 키워드 매칭

시험장에서는 다음의 영어 단어(What, How) 매칭만 기억하면 1초 만에 정답을 고를 수 있습니다!

* **데이터 관점:** 업무가 **어떤 데이터(What)**와 관련이 있는지
* **프로세스 관점:** 업무가 **무엇을(What) 해야 하는지, 또는 어떻게(How)** 하는지
* **상관 관점:** 프로세스에 의해 **데이터가 어떻게(How) 영향**을 받는지 (CRUD)

> **📝 기출 예시:** "데이터 모델링의 관점 중, 업무가 실제로 하고 있는 일은 무엇인지 또는 무엇을 해야 하는지를 모델링하는 관점은?"
> **정답:** 프로세스 관점
